---
title: "Get Patient Settings"
description: "Returns aggregated monitoring settings for a patient based on all cohorts they belong to. 
When a patient belongs to multiple cohorts with conflicting settings, the least restrictive option is applied (logical OR). 
For cadence settings, the shortest interval is applied. For more information, see the [guide about Cohorts](/medical-api/handling-data/cohorts)"
api: "GET /medical/v1/patient/{id}/settings"
---

## Path Params

<ParamField path="id" type="string" required>
  The ID of the patient whose settings are to be returned.
</ParamField>

## Response

<ResponseField name="settings" type="object" required>
  The aggregated monitoring settings for the patient based on their cohort memberships.
  <Expandable title="Settings properties">
    <ResponseField name="monitoring" type="object" optional>
      <ResponseField name="hie" type="string" optional>
        HIE-based document monitoring - subscribes patients to HIE-based document monitoring (non-ADT) / medical record refresh based on the cadence.
        - `never` | `monthly` | `biweekly` | `weekly` - Enable HIE data pulls at scheduled dates
        
        _Default: `monthly`_
      </ResponseField>
      <ResponseField name="adt" type="boolean" optional>
        ADT (Admission, Discharge, Transfer) monitoring - subscribes patients to ADT notifications based on their [primary address](/medical-api/api-reference/patient/create-patient#param-is-primary).  
        - `true` | `false` - Enable ADT monitoring notifications
        
        _Default: `false`_
      </ResponseField>
      <ResponseField name="pharmacy" type="object" optional>
        Pharmacy / medication history monitoring - enables retail fill data monitoring where available.  
        - `notifications` (`boolean`) - Enable/disable pharmacy monitoring notifications
        - `schedule` (`never` | `monthly` | `biweekly` | `weekly`) - Enable pharmacy data pulls at scheduled dates
        
        _Default: `{ notifications: false, schedule: never }`_
      </ResponseField>
      <ResponseField name="laboratory" type="object" optional>
        Laboratory notifications - subscribes patients for ongoing notifications of new lab results.  
        - `notifications` (`boolean`) - Enable/disable laboratory monitoring notifications  
        - `schedule` (`never` | `monthly` | `biweekly` | `weekly`) - Enable laboratory data pulls at scheduled dates
        
        _Default: `{ notifications: false, schedule: never }`_
      </ResponseField>
      <Info>
        For **pharmacy** and **laboratory** monitoring, choose either `notifications` OR `schedule`, but not both. If `notifications` is `true`, `schedule` value is ignored.
      </Info>
    </ResponseField>
  </Expandable>
</ResponseField>

## Settings Aggregation Logic

When a patient belongs to multiple cohorts, their settings are aggregated from all cohorts they belong to. If there are conflicting settings between cohorts, the least restrictive option is applied (logical OR).

### Examples

**Patient in one cohort with ADT monitoring enabled:**
```json
{
  "settings": {
    "monitoring": {
      "adt": true
    }
  }
}
```

**Patient in multiple cohorts, all with ADT monitoring enabled:**
```json
{
  "settings": {
    "monitoring": {
      "adt": true
    }
  }
}
```

**Patient in multiple cohorts, one with ADT enabled, one disabled:**
```json
{
  "settings": {
    "monitoring": {
      "adt": true
    }
  }
}
```

**Patient in multiple cohorts, one with weekly HIE, one with monthly HIE:**
```json
{
  "settings": {
    "monitoring": {
      "hie": "weekly"
    }
  }
}
```
